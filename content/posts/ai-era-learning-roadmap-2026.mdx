AI 正在快速改变每个领域的工作方式。与其被动观望，不如主动把 AI 变成自己的杠杆——用 AI 放大学习效率、自动化重复工作、辅助决策。这篇文章记录我接下来要攻克的七个方向，以及每个方向的学习路径和 AI 加持方案。

## 一、利用 AI 做多 Offer 选择

### 场景

你同时拿到了拼多多、字节跳动、阿里的 offer——怎么选？拍脑袋容易被某个单项（比如总包数字）牵着走。系统性的做法是：**把每个 offer 的所有维度展开，填进同一个对比框架，让 AI 帮你加权打分 + 模拟推演。**

### 第一步：填写 Offer 信息表

每个 offer 填一张表。以下是实际案例对比：

| 维度 | 拼多多（上海） | 字节跳动（上海） | 阿里（杭州） |
|------|-------------|---------------|-----------|
| **岗位** | 后端开发（交易链路） | 后端开发（推荐系统） | 后端开发（电商中台） |
| **月 base** | 45k × 16 | 48k × 15 | 42k × 16 |
| **年现金总包** | 72 万 | 72 万 | 67.2 万 |
| **股票/RSU** | 无（P6 无股票） | 期权 4 年 80 万（行权价待定） | 股票 4 年 60 万（RSU） |
| **签字费/搬家费** | 5 万签字费 | 3 万搬家费 | 无 |
| **工作地点** | 上海（金虹桥） | 上海（漕河泾） | 杭州（西溪园区） |
| **工作时长** | 11-11-6（传说） | 10-9-5（相对正常） | 10-8-5（视项目） |
| **通勤/租房** | 上海租房 ~4k/月 | 上海租房 ~4k/月 | 杭州租房 ~2.5k/月 |
| **公积金** | 7% 全额 | 12% 全额 | 12% 全额 |
| **晋升周期** | 1-2 年（看业绩） | 1.5-2 年 | 2-3 年（较慢） |
| **技术栈** | Java/Go + 自研中间件 | Go/Python + 内部平台 | Java + Spring 全家桶 |
| **团队规模** | 核心交易组，8 人 | 推荐中台，15 人 | 电商中台，20+ 人 |
| **业务前景** | 交易链路核心，稳但成长空间有限 | 推荐系统需求爆发，AI 相关性强 | 中台化趋势，但阿里电商增速放缓 |
| **跳槽认可度** | ⭐⭐⭐（因工时争议） | ⭐⭐⭐⭐（TMD 阵营） | ⭐⭐⭐⭐（阿里系） |
| **个人兴趣匹配** | 🟡 一般 | 🟢 高（推荐系统 = AI 方向） | 🟡 一般 |

### 第二步：设定你的价值维度和权重

每个人看重的维度不同。**在让 AI 分析之前，先想清楚自己想要什么。**

| 维度 | 权重 | 说明 |
|------|------|------|
| 💰 现金收入 | 25% | 到手多少钱（base + 奖金） |
| 📈 长期财务回报 | 15% | 股票/期权实际价值、公司增长潜力 |
| 🧠 技术成长 | 20% | 技术栈是否匹配长期方向、能学到什么 |
| 🚀 职业发展 | 15% | 晋升速度、跳槽认可度、管理机会 |
| ⚖️ 工作生活平衡 | 15% | 工作时长、通勤、是否双休 |
| 🎯 兴趣匹配 | 10% | 做的事情是否喜欢、是否有成就感 |

> **注意：权重是自己定的。** 如果你现在缺钱，现金收入权重要调到 40%；如果你想转 AI 方向，技术成长权重调到 30%。不要用别人的权重。

### 第三步：让 AI 做多维度对比分析

把上面的表喂给 Claude Code 或 ChatGPT：

```text
我同时拿到了三个 offer，以下是详细信息：

[粘贴上面的 Offer 信息表]

我的个人权重：
- 现金收入 25%、长期财务 15%、技术成长 20%
- 职业发展 15%、工作生活 15%、兴趣匹配 10%

请帮我：
1. 按每个维度对三个 offer 打分（1-10 分）
2. 加权计算总分，输出决策矩阵
3. 分析每个 offer 的最大风险和最大机会
4. 给出你的推荐并解释理由
5. 如果我调整权重（比如把技术成长调到 35%），排名会怎么变？
```

### 第四步：AI 深度追问

第一轮 AI 输出结果后，继续追问：

```text
"字节的期权 4 年 80 万，行权价还没定——假设行权价比当前估值高 20%，
这部分实际价值可能缩水到多少？请按乐观/中性/悲观三种情况估算"

"如果我在拼多多干 2 年再跳槽，和字节干 2 年再跳槽，市场给出的薪资涨幅分别大概多少？"

"三家在 AI 方向上的投入和技术积累怎么样？这对我 3 年后转型 AI 有什么影响？"

"帮我把杭州的生活成本和上海做详细对比，包括房租、物价、通勤时间、落户政策"
```

### 第五步：模拟 3 年推演

最后一轮，让 AI 推演不同选择的 3 年后结果：

```text
假设我选了 [拼多多/字节/阿里]，3 年后：
- 我的技术能力会提升到什么水平？
- 跳槽出去大概能拿到什么薪资？
- 如果公司业绩下滑/我被裁了，Plan B 是什么？
- 如果公司上市/估值翻倍，我的收益会是多少？
请分别推演三个选择的 3 年发展路径和风险。
```

### AI 做不到但你必须要做的

| 只有你能判断的 | 为什么 AI 做不了 |
|-------------|---------------|
| 团队氛围和直属 leader 靠不靠谱 | AI 没面过你未来的 leader，只在公开信息层面 |
| 你对某个业务的真实兴趣 | 这是价值观问题，不是数据问题 |
| 家庭因素（父母、伴侣、孩子） | 你的个人约束条件 AI 不知道 |
| 风险偏好 | 有人愿意赌独角兽期权，有人只要现金落地 |

**核心原则：AI 负责量化分析和推演，你负责加入"不可量化"的因素做最终决策。**

### 行动清单

- [ ] 建立自己的 Offer 对比模板（Excel or Notion）
- [ ] 每次跳槽季先把所有 offer 填入模板
- [ ] 定好个人权重（不能跳过这一步）
- [ ] 走完"AI 打分 → 追问 → 3 年推演"三轮分析
- [ ] 和 2-3 个业内朋友聊（AI 的盲区，人来补）

## 三、学习英语（旅游场景）

### 目标

出国旅游时能独立沟通——点餐、问路、酒店入住、购物、应急交流。**不求学术英语，只求实用生存英语。**

### 学习策略：场景驱动，而非语法驱动

不要从头啃语法书。直接学 10 个高频旅游场景的对话模板：

| 场景 | 核心句型 |
|------|---------|
| 机场/出入境 | Where is the baggage claim? / I'm here for tourism. |
| 酒店入住 | I have a reservation. / What time is checkout? |
| 餐厅点餐 | Can I have the menu? / I'll have this one. / The bill, please. |
| 问路 | How do I get to ___? / Is it walking distance? |
| 购物 | How much is this? / Can I get a tax refund? |
| 公共交通 | Which line goes to ___? / A one-way ticket, please. |
| 看病/药房 | I need something for a headache. / Is there a pharmacy nearby? |
| 紧急情况 | Can you help me? / I need to contact the embassy. |
| 闲聊 | Where are you from? / Any recommendations for things to do here? |
| 数字/时间 | How long does it take? / What time does it open? |

### AI 辅助方案

| 方法 | 具体操作 |
|------|---------|
| AI 口语陪练 | 用 ChatGPT 语音模式模拟旅游对话——让它扮演餐厅服务员、酒店前台、海关官员 |
| 场景对话生成 | 让 AI 为你生成特定目的地（如日本、泰国、欧洲）的定制化对话脚本 |
| 发音纠正 | 对着 AI 读句子，让它标出发音不准的单词 |
| 菜单翻译 | 出国前把目的地常见菜单喂给 AI，生成中英对照清单 |
| 每日一句 | 每天让 AI 推送一句旅游场景的实用英语，附带发音要点 |

### 行动清单

- [ ] 每天 15 分钟 AI 语音对话练习
- [ ] 出国前 2 周集中准备目的地专属场景对话
- [ ] 收集 300 个旅游核心词汇（食物、交通、住宿、数字、身体部位）

## 五、每日面试技能加强

### 目标

保持面试战斗力——随时能面、面了能过。覆盖三个方向：**LeetCode 算法**、**Java 基础与进阶**、**Android 体系**。

### 每日打卡结构

```
早晨 30min：LeetCode 1 题（理解 + 手写 + 复杂度分析）
午休 20min：Java/Android 知识点 1 个（深挖到底）
晚上 20min：系统设计 or 项目复盘
```

### LeetCode 刷题路线

不追求刷题数量，追求**模式识别**。按题型分类刷：

| 优先级 | 题型 | 代表题目 | 目标 |
|--------|------|---------|------|
| 🔴 必刷 | 数组/字符串/哈希表 | Two Sum, Group Anagrams, Longest Substring | 10 题/类 |
| 🔴 必刷 | 链表/栈/队列 | Reverse Linked List, Valid Parentheses, LRU Cache | 8 题/类 |
| 🔴 必刷 | 树/二叉树/二叉搜索树 | Max Depth, Validate BST, Level Order | 10 题/类 |
| 🟡 高频 | 动态规划 | Climbing Stairs, Coin Change, LCS | 15 题 |
| 🟡 高频 | 二分查找/双指针/滑动窗口 | Binary Search, 3Sum, Minimum Window Substring | 10 题 |
| 🟢 进阶 | 图/BFS/DFS/回溯 | Number of Islands, Permutations, Course Schedule | 10 题 |
| 🟢 进阶 | 堆/前缀树/并查集 | Top K Frequent, Trie Implementation | 5 题 |

## 六、AI 自动生成视频

### 目标

把博客内容自动变成视频——一篇文章 → 一个视频。不需要每次手动剪辑。

### 目前已有的能力

用 HyperFrames + Claude Code，我已经实现了：
- 写 HTML 就能出视频（GSAP 动画 + FFmpeg 渲染）
- macOS 本地中文 TTS 配音（Flo 语音）
- 9:16 竖屏和 16:9 横屏两种格式
- 数据可视化动画（柱状图、数字滚动）

详见我的 [HyperFrames 深度文章](/blog/hyperframes-claude-code)。

### 下一步：自动化流水线

理想状态是：**一篇新博客发布 → 自动生成对应的视频 → 发布到视频平台**。

```
新博客文章（MDX）
      ↓
  AI 提取核心论点 + 数据 + 金句
      ↓
  生成视频脚本（8-10 个场景）
      ↓
  生成 TTS 配音（Flo）
      ↓
  HyperFrames 渲染 HTML → MP4
      ↓
  自动发布到 YouTube / 小红书 / TikTok
```

### 要解决的问题

| 问题 | 难度 | 方案 |
|------|------|------|
| 从博客自动提取关键信息 | 🟢 低 | Claude API 总结文章 → 提取 8 个核心数据点 + 3 个金句 |
| 自动生成视频脚本 | 🟡 中 | 用固定模板 + AI 填充内容（标题、数据、结论三段式） |
| 自动匹配视觉风格 | 🟡 中 | 预设 3 套主题（科技/财经/生活），AI 根据文章分类选择 |
| TTS 自动配音 | 🟢 低 | 已解决，macOS `say` 命令 + Flo 语音 |
| 渲染自动化 | 🟢 低 | `npx hyperframes render` 一键渲染 |
| 自动发布 | 🟠 中高 | 需要各平台的 API（YouTube Data API v3 等） |

### 行动清单

- [ ] 设计一套固定的视频模板（标题卡片 + 数据卡片 + 结论卡片）
- [ ] 写一个脚本：输入 MDX 文件路径 → 输出 MP4
- [ ] 对接至少一个视频平台的发布 API

## 七、AI 运营小红书全自动化

### 目标

小红书账号运营全流程 AI 化——**自动生成内容 → 自动发布 → 自动收集热点 → 自动复盘优化**。人只做最终审核。

### 全自动化工作流

```
热点监控（AI 每日抓取）
       ↓
选题决策（AI 筛选 + 人去重/确认）
       ↓
内容生成（AI 写文案 + 做图 + 生成视频）
       ↓
定时发布（API 自动发布）
       ↓
数据回收（评论/点赞/收藏/转发）
       ↓
自动复盘（AI 分析哪些选题/标题/发布时间效果好）
       ↓
策略调整（优化选题方向 + 发布时间 + 标题风格）
```

### 各环节详解

#### 1. 自动收集热点

| 来源 | 抓取方式 | 频率 |
|------|---------|------|
| 小红书热搜/发现页 | Playwright 截图 + AI 识别 or 手动 RSS 替代方案 | 每 4 小时 |
| 抖音热搜 | 抖音热榜 API or 第三方聚合 | 每 6 小时 |
| 微博热搜 | 微博 API or 爬虫 | 每 4 小时 |
| 知乎热榜 | 知乎 API | 每 6 小时 |
| 目标赛道关键词 | Google Trends + 5118/5118 替代品 | 每天 |

技术栈：**Python 脚本 + Playwright（截图） + Claude API（分析截图提取热点）→ JSON 热点库**

#### 2. AI 筛选选题 + 人去重确认

AI 对抓取的热点进行打分：

| 维度 | 权重 | 说明 |
|------|------|------|
| 与我账号定位的相关度 | 40% | 是否在目标赛道内 |
| 话题热度 | 30% | 搜索量/讨论量趋势 |
| 竞争度 | 20% | 已有内容是否饱和 |
| 可执行性 | 10% | 我能不能快速产出内容 |

AI 自动出 5 个选题建议 → 人挑 1-2 个 → AI 开始生产内容。

#### 3. AI 生成内容

| 内容组件 | AI 工具 | 产出 |

| 标题 | Claude/GPT | 5 个备选标题（含 emoji、数字、悬念等钩子） |

#### 4. 自动发布

小红书的自动发布比较特殊，目前没有官方公开 API。可行方案：

| 方案 | 优点 | 缺点 |
|------|------|------|
| iPhone 快捷指令 + 定时触发 | 免费，官方途径 | 功能有限 |
| 第三方工具（如零克、灰豚等） | 功能全 | 收费，有封号风险 |

## 八、基金股票投资：低位吸盘纪律

### 核心策略

**比高点低于 10% 就开始分批进场，回到高点附近就卖出。** 这是一种"震荡市网格交易"策略，核心是**纪律 > 判断**。

### 策略逻辑

```
                     ← 高点 B（卖出）
    ████            |
   ██  ██           | ← 卖出区间（高点 -0~5%）
  ██    ██          |
 ██      ██         |
██        ██        |
           ██       |
            ██      | ← 观察区间（不做操作）
             ██     |
              ██    |
               ██   | ← 买入区间（高点 -10%~-15%）
                ██  |
                 ██ | ← 低点 A（买入）
```

### 操作规则

| 条件 | 操作 | 仓位 |
|------|------|------|
| 价格跌至高点的 90% 以下（-10%） | **开始进场**，买入第一批 | 计划仓位的 25% |
| 继续下跌至高点的 85%（-15%） | **加仓**第二批 | 再加 25% |
| 继续下跌至高点的 80%（-20%） | **加仓**最后一批 | 再加 50%（满仓） |
| 价格上涨至高点的 95%（-5%） | **开始出货**，卖出第一批 | 卖出 40% |
| 价格回到高点 100% | **全部清仓** | 卖出剩余 60% |

**关键纪律：**
- 到点就操作，不犹豫。不要想"会不会继续跌/涨"
- 不在 -10% 以内买入，不在 0~-5% 观望区间操作
- 跌破 -20% 停止加仓，重新评估。可能趋势变了
- 每次操作的仓位比例提前定好，盘中不临时改

### 适用品种

| 品种 | 特点 | 历史波动率参考 | 适合本策略？ |
|------|------|-------------|------------|
| 黄金（ETF/实物） | 避险资产，震荡区间明显 | 15-25% | ✅ 很适合 |
| 纳指 ETF（QQQ） | 长牛 + 中期回调 | 20-30% | ✅ 适合 |
| 沪深 300 ETF | A 股代表，震荡频繁 | 25-30% | ✅ 很适合 |
| A 股个股 | 波动大，个别风险高 | 30-60% | ⚠️ 慎用，需选龙头 |
| 港股科技基金 | 波动极大，趋势性强 | 30-50% | ⚠️ 注意区分震荡和趋势转换 |
| 美债 ETF | 低波动，不适合 10% 区间 | 5-10% | ❌ 需要调整参数 |

### AI 辅助清单

| 功能 | 实现方式 |
|------|---------|
| 高点追踪 | AI 定期抓取 52 周高点，计算当前价格相对高点的百分比 |
| 买入/卖出提醒 | 当价格触及 -10%/-15% 时自动推送通知 |
| 复盘分析 | 每次操作后 AI 记录：操作日期、价格、理由、结果，每月生成复盘报告 |
| 情绪纪律 | 让 AI 在你冲动时泼冷水——"现在价格只跌了 5%，还没到你定的 -10% 买入区间，建议等待" |
| 多品种监控 | AI 同时追踪 10+ 个目标品种，按偏离高点的幅度排序，帮你定位最佳品种 |
| 周报 | 每周自动生成：各品种偏离幅度、操作记录、持仓盈亏、下周关注点 |

这些功能完全可以写成一个 **Claude Code + TaskCreate + Cron** 的本地自动化系统，详见我之前的 [股票数据面板](/market) 实践。
