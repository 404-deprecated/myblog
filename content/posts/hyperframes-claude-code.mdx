## 一句话说清楚

HyperFrames 是 HeyGen 开源的**视频渲染框架**。它的核心思路是：**你写 HTML，它出 MP4**。没有 React，没有 DSL，没有构建步骤——一个 `index.html` 文件在浏览器里能播，用命令行就能渲染成视频。

更重要的是，它**专门为 AI 编程助手（Claude Code、Cursor、Gemini CLI）设计了技能系统**，让 AI 能像人类视频剪辑师一样理解你的需求、写出正确的视频代码。

---

## 为什么需要它？Remotion 的替代品

如果你用过 Remotion（React 写视频），你应该知道它的痛点：

| 痛点 | Remotion | HyperFrames |
|------|----------|-------------|
| 写一个视频 | React 组件 + 构建工具链 | 一个 HTML 文件 |
| 改一个文字 | 改 JSX → 重新构建 → 等编译 | 改 HTML → 刷新浏览器 |
| 授权 | 公司规模收费（非 OSI 开源） | **Apache 2.0**，完全开源 |
| AI 写视频代码 | AI 对 React 视频组件容易出错 | AI 天生会写 HTML + CSS |
| 渲染确定性 | 依赖 wall-clock 时间 | **帧精确、可 seek**，同输入=同输出 |

HyperFrames 把视频制作的门槛降到了"会写网页就会做视频"的程度。对于 AI 来说，这比写 React 组件要准确得多——因为 AI 模型见过的 HTML/CSS 训练数据数量级远超 Remotion 组件。

---

## 架构：三个核心模块

```
你写的 HTML（data-* 属性标记场景和动画）
        ↓
   Puppeteer（无头 Chrome 逐帧截图）
        ↓
    FFmpeg（编码 + 音轨混合）
        ↓
      MP4 视频文件
```

**关键设计：**

1. **HTML 原生创作**：视频片段用 `data-start`、`data-duration`、`data-track-index` 属性标记
2. **帧适配器模式**：支持 GSAP、Anime.js、Lottie、Three.js、CSS Animations、WAAPI 六种动画引擎
3. **确定性渲染**：禁止 `Math.random()`、`Date.now()`、`setInterval` 等非确定性 API，确保每次渲染结果完全一致
4. **无构建步骤**：HTML 在浏览器里直接预览，`npx hyperframes preview` 开热重载

## 安装与使用

### 前置条件
- Node.js ≥ 22
- FFmpeg 已安装

### 最简流程

```bash
# 1. 创建项目
npx hyperframes init my-video

# 2. 浏览器预览（热重载）
npx hyperframes preview

# 3. 渲染输出 MP4
npx hyperframes render
```

### 安装组件库

HyperFrames 有一个 **50+ 现成组件**的 Catalog 系统：

```bash
npx hyperframes add flash-through-white   # 着色器转场
npx hyperframes add instagram-follow      # 社交媒体叠加层
npx hyperframes add data-chart            # 动画图表
```

## 和 Claude Code 结合：这才是重点

HyperFrames 最特别的地方是**为 AI 编程助手设计了 15 个技能（Skills）**，并且自带 Claude Code 插件。

### 1. 安装技能

```bash
npx skills add heygen-com/hyperframes
```

安装后，Claude Code 会自动加载这些斜杠命令：

| 命令 | 用途 |
|------|------|
| `/hyperframes` | 创建视频组合（composition） |
| `/hyperframes-cli` | 开发命令（init, lint, preview, render） |
| `/hyperframes-media` | 素材预处理（TTS 语音合成、Whisper 转写、背景去除） |
| `/tailwind` | Tailwind CSS v4 样式支持 |
| `/gsap` | GSAP 动画时间线 |
| `/animejs` | Anime.js 动画 |
| `/css-animations` | CSS 关键帧动画 |
| `/lottie` | Lottie 动画 |
| `/three` | Three.js 3D 场景 |
| `/waapi` | Web Animations API |

### 2. Claude Design + Claude Code 双阶段工作流

HyperFrames 把视频制作拆成两个阶段：

**第一阶段：Claude Design（品牌 + 布局 + 初版动画）**

下载设计指南文件，在 Claude Design 对话中附加，让 AI 产出一个结构正确的初稿：

> "为我的 SaaS 产品做一个 15 秒的产品介绍视频，深色主题，科技感风格"

Claude Design 会输出：
- 品牌配色、字体选择（注意它禁止使用 Inter/Roboto/Poppins 等"AI 默认字体"）
- 6-8 个场景的布局和文案
- 关键位置的 shader 转场选择（不是每个场景都加转场！每 6-8 个场景用 2 个转场最合适）
- 符合数据属性规范的 HTML 结构

**第二阶段：Claude Code（细化时间、缓动、节奏、QA）**

```bash
# 在 Claude Code 中继续迭代
"把标题字号放大 2 倍，切换到暗色模式，结尾加一个淡出"
"在 0:03 加一个字幕条，显示我的名字和职位"
"把第 3 个场景的入场动画改成从左侧滑入"
```

### 3. 实战：用自然语言描述就能生成视频

**冷启动——直接描述视频需求：**

```
/claude 创建一个 10 秒的产品介绍视频：
- 背景是一个科技感的粒子动画
- 标题"Introducing HyperFrames"在 1 秒时淡入
- 在 3 秒时出现"Write HTML. Render Video."副标题
- 在 7 秒时出现 CTA 按钮"Get Started"
- 全程有背景轻音乐
- 输出 1920x1080 的 MP4
```

**温启动——把现有素材变成视频：**

```
/claude 把这个 CSV 文件做成一个动画柱状图竞赛视频
/claude 总结这个 PDF 报告，生成 45 秒的电梯演讲视频
/claude 抓取这个网页，做成带旁白的介绍视频
```

**迭代——把 AI 当视频剪辑师用：**

```
"这个场景太快了，延长到 4 秒"
"把字体从 Inter 换成 DM Sans"
"在 Logo 上加一个呼吸浮动效果"
```

### 4. 自动化流水线

因为 HyperFrames 的渲染是**完全确定性**的（同输入=同输出），它非常适合 CI/CD 场景：

```yaml
# GitHub Actions 示例思路
- name: 生成产品演示视频
  run: |
    npx hyperframes render --input ./compositions/demo.html --output ./output/demo.mp4
```

你可以在每次发布新版本时自动生成最新的产品演示视频，或者根据数据源自动生成每日数据简报视频。

## 进阶：API 驱动的视频后期全自动化

除了用 HTML 从零创建视频，HyperFrames 还提供了 **REST API**，可以和 Claude Code 配合实现**视频后期制作的全自动化**——给一段原始视频自动加片头、字幕条、同步字幕，全程不需要打开时间线编辑器。

MindStudio 的工程师团队总结了一套可复现的工作流，核心思路是：**视频后期中大量机械性工作（加字幕条、对齐字幕、品牌叠加层）本质上是"遵循规则的体力活"，恰好是 AI Agent 最擅长的事。**

### 整体架构

```
用户用自然语言描述需求
        ↓
Claude Code 解析为结构化 API 调用
        ↓
HyperFrames API 渲染动画元素
        ↓
Claude Code 将所有图层合成最终视频
```

Claude Code 的角色是**翻译官 + 调度器**——把"在第 32 秒加一个产品总监的字幕条"翻译成 API 参数，然后调用 HyperFrames 渲染，最后把所有图层合成。HyperFrames 的角色是**渲染引擎**——接收结构化指令（元素类型、时间码、样式、位置），返回渲染好的视频图层。

### 第一步：定义三个核心技能

整个流程建立在三个 Claude Code 技能之上，每个技能只做一件事，可以像乐高一样组合：

| 技能 | 输入 | 输出 |
|------|------|------|
| `generate-motion-graphic` | 描述文本 + 时间码 + 品牌风格 JSON | 渲染好的片头/动画片段 |
| `add-animated-overlay` | 元素类型 + 文字 + 时间码 + 位置 | 叠加层视频 |
| `sync-captions` | 转录文本 + 字幕样式 | 同步字幕轨道 |

这三个技能的巧妙之处在于**每个都很窄（well-scoped）**——不是一个"做所有后期"的庞然大物，而是三个可以独立调用、自由组合的原子能力。

### 第二步：用 Claude Code 搭建 HyperFrames API 客户端

```text
"写一个 HyperFrames API 客户端，要求：
- 从环境变量读取 API Key 进行认证
- 暴露三个函数：创建元素、创建合成、触发渲染
- 支持指数退避的速率限制处理
- 返回结构化的响应，包含渲染后的素材 URL"
```

Claude Code 生成一个封装了三个核心端点的客户端：
- `/elements` — 创建文字、形状、图片元素并附加动画属性
- `/compositions` — 将多个元素组合到一条时间线
- `/render` — 触发渲染，返回文件下载链接

### 第三步：生成动态片头

```text
"生成一个 4 秒的片头动画：
- 公司名 'Northlight' 从左侧淡入，深色背景
- 标语 'Built for speed' 随后在下方出现
- 使用 /assets/brand.json 的品牌风格
- 放在视频开头"
```

Claude Code 的内部处理链路：解析"从左侧淡入"→ 映射为 `slide-in-left` + 透明度过渡 → 从品牌 JSON 提取字体/颜色/间距 → 创建合成 → 触发渲染 → 返回文件路径。

### 第四步：在指定时间码添加动画叠加层

**单个叠加层：**

```text
"在 0:32 添加一个字幕条，显示 'Sarah Chen, Head of Product'，
白字、半透明深色背景条，从左侧滑入，保持 4 秒后滑出，
使用标准字幕条位置（画面底部 20% 区域）"
```

**批量叠加层——一条 prompt 处理整批：**

```text
"为以下时间戳添加演讲者字幕条，统一风格（白字、深色半透明底、左滑入、保持 3 秒）：
- 0:12 — 'Marcus Webb, CEO'
- 1:45 — 'Priya Nair, Engineering'
- 3:20 — 'David Lim, Sales'"
```

Claude Code 将这些处理为循环，依次调用 API，返回所有叠加层文件。**传统后期中这需要 20-30 分钟一个标签的手工操作，现在一条 prompt 几分钟全部搞定。**

### 第五步：同步字幕

```text
"为附件的转录文本生成同步字幕轨道：
- 逐句显示（非逐词）
- 白字、加粗、居中、画面下方三分之一处
- 淡入淡出过渡（200ms）
- 使用 brand.json 中的品牌字体
- 导出为合成视频图层"
```

处理流程：将转录文本分句 → 映射起止时间码 → 为每句创建 HyperFrames 文字元素 → 构建按序播放的合成。

### 第六步：串联成完整流水线

```text
"对 /videos/product-launch-raw.mp4 进行后期处理：
1. 用 brand.json 生成 3 秒片头
2. 根据 timestamps.csv 添加演讲者字幕条
3. 根据 transcript.srt 生成同步字幕
4. 将所有图层合成到一个输出文件
输出到 /videos/product-launch-final.mp4"
```

执行顺序：技能一 → 片头文件 → 技能二 → 叠加层 → 技能三 → 字幕轨 → 最终合成步骤将四个图层（源视频 + 片头 + 叠加层 + 字幕）合并。

### 审核环节

两种方式加入人工审核 gate：
- **Slack 集成**：Claude Code 发预览链接到 Slack，团队成员回复 "approve" 或 "revise"
- **条件逻辑**：渲染后在合成前暂停，等人工确认

### 常见坑与解法

| 问题 | 解法 |
|------|------|
| 时间码超出视频时长 | 生成叠加层前先校验视频时长 |
| 品牌 JSON 缺字段 | 定义 required schema，加载时验证 |
| 长句字幕换行不自然 | 设 40-50 字上限，自动再分段 |
| 渲染队列延迟 | 轮询检查渲染状态，每 10 秒一次直到完成 |
| 多语言字体缺失 | 确保品牌字体包含目标语言字符集 |

### 这个工作流适合谁？

| 团队类型 | 场景 | 收益 |
|---------|------|------|
| 市场团队 | SaaS 产品演示视频，需要跨大量素材保持品牌一致 | 品牌元素一次配置，重复使用 |
| 内容创作者 | YouTube/博客视频，自动加字幕和品牌片头 | 字幕 + 片头自动化 |
| 客户成功 | 客户证言视频，需要批量加字幕条 | 一个季度的证言视频一次跑完 |
| 代理商 | 规模化视频制作，不增加人头 | 程序化生产替代手工操作 |

**这个流水线的核心价值：** 传统后期中"每个视频都要重复做一遍"的机械工作——加字幕条、对齐字幕、品牌叠层——全部变成一条自然语言指令。视频编辑花一个下午做的事，Agent 几分钟跑完。代价是前期花时间搭好技能和品牌配置文件，但这是"一次配置、持续复用"的投资。

> 本节工作流参考了 [MindStudio 的 AI Video Editing 文章](https://www.mindstudio.ai/blog/ai-video-editing-claude-code-hyperframes)，其中详细描述了基于 HyperFrames API 的后期自动化方案和 Remy 产品化封装思路。

HyperFrames 的 `/hyperframes-media` 技能提供三个本地 AI 模型，完全离线运行，不消耗任何 API 额度：

| 功能 | 底层模型 | 用途 |
|------|---------|------|
| TTS 文字转语音 | Kokoro（本地） | 生成旁白配音 |
| 语音转文字 | Whisper（本地） | 自动字幕/转写 |
| 背景去除 | u2net（本地） | 人像抠图 |

这一步**零成本 + 数据不出本机**，特别适合需要频繁生成视频的团队。

## 和 Remotion 的对比：该选哪个？

| 维度 | HyperFrames | Remotion |
|------|-------------|----------|
| **创作格式** | HTML + CSS + 动画库 | React 组件（TSX） |
| **构建步骤** | 无，HTML 直接播 | 需要打包 |
| **AI 友好度** | ⭐⭐⭐⭐⭐ AI 天然擅长 HTML | ⭐⭐⭐ React 组件易出错 |
| **动画模型** | 帧精确、可 seek | Wall-clock 驱动 |
| **已有 HTML/CSS** | 直接粘贴用 | 必须重写成 JSX |
| **分布式渲染** | 单机 | 支持 Lambda |
| **授权** | Apache 2.0（真开源） | 源码可见，商业收费 |
| **社区生态** | 新兴，50+ 组件 | 成熟，大量模板 |

**选 HyperFrames 如果你：** 想要真开源、AI 辅助创作、简单到 HTML 就能搞定、CI/CD 自动化视频。

**选 Remotion 如果你：** 已有 React 团队、需要分布式渲染、社区模板够用不介意授权费。

## 总结

HyperFrames 把视频制作的抽象层级从"React 组件"降到了"HTML 文件"，这个降维恰好命中了 AI 编程的甜点区——AI 写 HTML/CSS 的准确率远高于 React 视频组件。配合 15 个 Claude Code 技能、本地 AI 媒体处理、确定性渲染引擎，它让"用自然语言生成视频"从 demo 变成了生产力工具。

对于已经在用 Claude Code 做开发的团队来说，安装一个 `npx skills add` 就能让同一个 AI 助手同时写代码和做视频——这个效率提升是立竿见影的。
