这篇文章只列真正在用的工具——判断标准是「这周用过至少三次」。那些体验了一次就再也没打开的，不在这里。

## 编码

### Claude Code

目前主力 AI 编码工具。对复杂上下文的理解比其他工具强，在处理「改这段逻辑，但不要破坏那边的接口」这类需求时表现明显更好。

缺点是贵。但如果算上省下来的 debug 时间，对我来说还是值的。

### GitHub Copilot

补全用。和 Claude Code 共存，两者分工不一样——Copilot 处理行内补全，Claude Code 处理需要上下文的大块改动。

## 写作 / 思考

### ChatGPT

用来做「橡皮鸭调试」——把一个想法或问题说给它听，不一定要它给出答案，只是把思路捋清楚。

也用来起草邮件、PR 描述、技术文档初稿。

### NotebookLM

把几篇论文或长文档扔进去，让它做摘要、提问、对比。阅读英文技术资料时省了不少时间。

## 图像

### Midjourney

主要用来做 PPT 配图和博客封面。不追求写实，偏向插图风格。

---

## 没在用的（以及原因）

- **Cursor**：和 VS Code + Copilot + Claude Code 的组合相比，没有明显优势，反而多一套配置要维护。
- **Perplexity**：搜索结果质量有时不稳定，遇到技术问题还是更信 Stack Overflow + 官方文档。
- **各种套壳 GPT 应用**：大部分没有比直接用原版多什么，付两份钱没必要。

---

工具是手段，不是目的。每隔一段时间我都会重新审视这个清单，把真正有价值的留下来。
